# Behavior Imports

## Initial imports (2026-02-13)

### Carmack
- Behavior: Ship one hard thing before noon.
- Why this helps me: It forces meaningful progress early before noise and context switching take over.
- Daily check: Did I ship one genuinely hard thing before 12:00 PM?

### Hopper
- Behavior: Challenge one default assumption each day.
- Why this helps me: It prevents stale thinking and keeps me from blindly inheriting broken defaults.
- Daily check: What assumption did I challenge today, and what changed because of it?

### Eno
- Behavior: Use constraints first when stuck.
- Why this helps me: Constraints reduce overwhelm and create fast creative traction.
- Daily check: When I got stuck, did I define constraints before expanding options?

### McNulty
- Behavior: Do one invisible support task well.
- Why this helps me: Quiet reliability compounds trust and system health over time.
- Daily check: What behind-the-scenes support task did I complete with care today?

### Molnár
- Behavior: Explore one small rule deeply before adding complexity.
- Why this helps me: Depth on a simple rule produces stronger foundations than premature complexity.
- Daily check: Which small rule did I explore deeply before adding anything new?

### Knuth
- Behavior: Improve one sentence/line for clarity every day.
- Why this helps me: Daily clarity edits improve thinking quality and communication precision.
- Daily check: Which sentence/line did I make clearer today?
